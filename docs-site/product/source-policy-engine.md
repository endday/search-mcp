# Source Policy Engine

原文位于仓库中的 `docs/source-policy-engine-dev.md`。

这份文档定义的是一个来源策略层，不是搜索引擎，也不是问答系统。

## 它解决的问题

Agent 在发起检索前，通常缺少两件事：

- 不知道优先去哪里找
- 不知道哪些来源应该避开

SPE 的目标就是把“选源决策”做成稳定接口。

## 放进文档站后的价值

- 更适合跟搜索网关主线做概念分层
- 后续可以和版本路线图、产品草案互相链接
- 能逐步演化成正式的设计文档，而不是孤立 markdown

## 建议后续拆分

- 定位与边界
- 接口定义
- 数据组织方式
- 与搜索系统的集成
