# AI Context Resolver 草案

原文位于仓库中的 `docs/chat.md`。

这是一个更偏产品草案的文档，和 Search MCP 主线相关，但不完全等同于当前仓库的运行时能力。

## 建议如何放置

它适合被定义成“探索性产品文档”：

- 不直接当成当前功能说明
- 也不应该继续散落在 `docs/` 根目录里
- 放在 Product 区域，明确它是方向草案而不是既有能力

## 当前可读结论

它的主张是：

- 不做大而全知识库
- 先做来源与上下文入口层
- 让 Agent 在检索前知道“该去哪看”

这个方向和 Source Policy Engine 在概念上是能衔接的。
