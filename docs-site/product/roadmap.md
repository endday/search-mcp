# 版本路线图

原文位于仓库中的 `docs/version-roadmap.zh.md`。

当前最重要的路线判断其实已经比较明确：

- 这个项目不是搜索网站
- 这个项目也不是 AI 问答系统
- 它更像一个可自托管、可观测、可策略化的 Agent Search Gateway

## 当前对外主线

后续版本应该优先强化：

- 可运维性
- 结果可解释性
- 搜索策略
- 管理和运营能力

## 下一步候选能力

- 增加 YouTube 搜索接入，优先基于 `yt-dlp` 落地
- 查询格式先采用 `ytsearchN:{query}`
- 首版只返回基础字段：标题、链接、频道、时长、播放量
- 补齐超时、错误处理和短时缓存
- 只有在性能成为明确瓶颈后，再评估自建 `youtubei` 搜索实现
- 增加 Exa 作为可选 Web search 上游
- 优先按 HTTP API provider 接入，不先走 MCP
- 通过环境变量管理 `EXA_API_KEY`
- 结果映射到现有统一搜索结果结构，并纳入现有排序/策略体系
- 明确它是付费或需密钥的增强上游，不替代默认免费搜索源

## 不该再继续放大的部分

- 首页展示层
- 以远程部署为核心的 MCP 使用路径
- 过重的“网站化”表达

## 为什么文档站重要

路线图、版本原则、阶段目标都属于持续演化内容。它们最适合放在 VitePress 里，不适合长期滞留在 README 或 Worker 首页。
