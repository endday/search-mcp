# 版本路线图

原文位于仓库中的 `docs/version-roadmap.zh.md`。

当前最重要的路线判断其实已经比较明确：

- 这个项目不是搜索网站
- 这个项目也不是 AI 问答系统
- 它更像一个可自托管、可观测、可策略化的 Agent Search Gateway

## 当前对外主线

后续版本应该优先强化：

- 可运维性
- 结果可解释性
- 搜索策略
- 管理和运营能力

## 不该再继续放大的部分

- 首页展示层
- 以远程部署为核心的 MCP 使用路径
- 过重的“网站化”表达

## 为什么文档站重要

路线图、版本原则、阶段目标都属于持续演化内容。它们最适合放在 VitePress 里，不适合长期滞留在 README 或 Worker 首页。
